## 1. 构件分类体系概述

本系统采用四级分类体系对房屋构件进行组织和管理：

| 层级 | 说明 | 示例 |
|------|------|------|
| 一级分类 | 构件大类，按结构功能划分 | 地基基础、上部承重结构、围护结构、其他 |
| 二级分类 | 具体构件类型 | 混凝土柱、砖墙、屋架等 |
| 损坏查勘情况 | 构件的具体损坏状态描述 | 有裂缝、风化腐蚀、倾斜变形等 |
| 评定结果 | 损坏程度等级 | 完好、基本完好、一般损坏、严重损坏、危险点 |

### 一级分类列表

| 序号 | 一级分类 | 二级分类数量 | 评定标准条数 |
|------|----------|-------------|-------------|
| 1 | 地基基础 | 2 | 19 |
| 2 | 上部承重结构 | 7 | 122 |
| 3 | 围护结构 | 6 | 88 |
| 4 | 其他 | 12 | 104 |
| **合计** | **4** | **27** | **333** |

## 2. 评定结果等级说明

| 评定结果 | 等级代码 | 说明 |
|----------|----------|------|
| 完好 | a | 构件状态良好，无损坏 |
| 基本完好 | b | 构件有轻微损坏，不影响使用功能 |
| 一般损坏 | c | 构件有明显损坏，需要维修 |
| 严重损坏 | d | 构件损坏严重，影响安全使用 |
| 危险点 | e | 构件处于危险状态，必须立即处理 |

## 7. 数据库表结构

### 7.1 评定标准表 (evaluation_standard)

```sql
CREATE TABLE evaluation_standard (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  category VARCHAR(100) NOT NULL,          -- 一级分类
  component_type VARCHAR(100) NOT NULL,    -- 二级分类（构件类型）
  description TEXT NOT NULL,               -- 损坏构件查勘情况描述
  evaluation_result VARCHAR(255),          -- 评定结果
  evaluation_clause VARCHAR(255),          -- 评定标准条款
  sort_order INTEGER DEFAULT 0,            -- 显示排序
  _created_at TIMESTAMP(3) WITH TIME ZONE NOT NULL DEFAULT CURRENT_TIMESTAMP,
  _created_by user_profile,
  _updated_at TIMESTAMP(3) WITH TIME ZONE NOT NULL DEFAULT CURRENT_TIMESTAMP,
  _updated_by user_profile
);

-- 索引
CREATE INDEX idx_evaluation_standard_category ON evaluation_standard(category);
CREATE INDEX idx_evaluation_standard_component_type ON evaluation_standard(component_type);
CREATE INDEX idx_evaluation_standard_sort ON evaluation_standard(sort_order);
```

### 7.2 构件模板表 (component_template)

```sql
CREATE TABLE component_template (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  category VARCHAR(100) NOT NULL,          -- 一级分类
  name VARCHAR(100) NOT NULL,              -- 构件类型名称
  check_items JSONB NOT NULL,              -- 检查项目列表
  display_order INTEGER DEFAULT 0,         -- 显示排序
  _created_at TIMESTAMP(3) WITH TIME ZONE NOT NULL DEFAULT CURRENT_TIMESTAMP,
  _created_by user_profile,
  _updated_at TIMESTAMP(3) WITH TIME ZONE NOT NULL DEFAULT CURRENT_TIMESTAMP,
  _updated_by user_profile
);
```

### 7.3 构件检查记录表 (component_check)

```sql
CREATE TABLE component_check (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  survey_id UUID NOT NULL,                 -- 关联鉴定记录
  name VARCHAR(100),                       -- 构件名称（二级分类）
  category VARCHAR(100),                   -- 构件分类（一级分类）
  axis_line VARCHAR(100),                  -- 轴线位置
  checked_item_ids JSONB DEFAULT '[]',     -- 勾选的评定标准ID列表
  ai_evaluation_result VARCHAR(255),       -- AI推理的评定结果
  ai_evaluation_clause VARCHAR(255),       -- AI推理的评定标准条款
  photos TEXT[],                           -- 照片URL列表
  _created_at TIMESTAMP(3) WITH TIME ZONE NOT NULL DEFAULT CURRENT_TIMESTAMP,
  _created_by user_profile,
  _updated_at TIMESTAMP(3) WITH TIME ZONE NOT NULL DEFAULT CURRENT_TIMESTAMP,
  _updated_by user_profile
);

CREATE INDEX idx_component_check_survey_id ON component_check(survey_id);
```

## 8. API接口规范

### 8.1 获取评定标准列表

```http
GET /api/evaluation-standards?category={category}&componentType={componentType}
```

**请求参数**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| category | string | 否 | 一级分类筛选 |
| componentType | string | 否 | 二级分类（构件类型）筛选 |

**响应示例**:
```json
{
  "items": [
    {
      "id": "uuid",
      "category": "上部承重结构",

      "evaluationResult": "一般损坏",
      "evaluationClause": "\"危标\"第3.3.1.2条",
      "sortOrder": 0
    }
  ]
}
```
