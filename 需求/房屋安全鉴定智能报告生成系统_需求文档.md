# 房屋安全鉴定智能报告生成系统 — 需求文档

## 1. 项目背景

本项目已有一套房屋安全鉴定小程序系统（React + NestJS + PostgreSQL），实现了鉴定记录管理、数据采集、AI 推理和报告导出的基础框架。现需基于**《房屋安全鉴定现场查勘表（砌体结构）》**的完整采集数据，自动生成**《房屋安全鉴定书》**正式报告。

核心流程：**现场查勘人员填写查勘表 → 系统自动推理 + 填充 → 生成正式鉴定书 Word 文档**。

## 2. 输入与输出定义

### 2.1 输入：《房屋安全鉴定现场查勘表（砌体结构）》

查勘表包含 28 个表格区域，按内容分为五大模块：

| 模块 | 覆盖字段 | 表索引 |
|------|---------|--------|
| **A. 委托及房屋概况** | 委托单位/联系人/电话、房屋名称、座落、建筑面积、层数、建成年份、结构类型、基础/墙体/楼面/屋面型式、原设计用途/现用途等 | Table 0 |
| **B. 历史变更与使用状况** | 用途变更、改造扩建、加固修缮、灾害、拆改结构、乱搭乱盖、超载使用 | Table 0（下半） |
| **C. 专项标识** | 是否优保建筑、农村危房改造、校外培训机构、自建房、经营性自建房、排查编码等 | Table 0（底部） |
| **D. 证件资料与周边环境** | 委托方提供资料清单、周边建设施工情况、现场草图 | Table 1 |
| **E. 房屋受损情况（按构件）** | 详细的构件损伤查勘数据，每种构件类型一个独立子表，包含查勘描述、评定结果、标准条款 | Tables 2–27 |

#### E 模块 — 构件分类与评定体系

查勘表采用"**逐项勾选法**"：每个构件类型下列出若干标准化的损坏情形描述，查勘人在现场勾选匹配的情形，每条情形预置了对应的**评定结果**和**标准条款引用**。

**评定结果等级（按严重程度）：**

| 等级 | 标准依据 | 含义 |
|------|---------|------|
| 完好 | 《房屋完损等级评定标准》3.1.x | 构件无损 |
| 基本完好 | 《房屋完损等级评定标准》3.2.x | 轻微瑕疵 |
| 一般损坏 | 《房屋完损等级评定标准》3.3.x | 需关注但非危险 |
| 严重损坏 | 《房屋完损等级评定标准》3.4.x | 需维修 |
| 危险点 | 《危险房屋鉴定标准》5.x.x | 存在安全隐患 |
| 非危险状态 | 《危险房屋鉴定标准》/《武汉市规程》 | 确认安全 |

**三级标准依据：**
- **"危标"** — 《危险房屋鉴定标准》（JGJ 125-2016）
- **"完标"** — 《房屋完损等级评定标准》（建设部城住字[84] 第678号）
- **"规程"** — 《武汉市房屋安全鉴定技术规程》（DB4201/T 537-2018）

**构件类型覆盖（共 18 类，333 条评定标准已在库中）：**

```
地基基础/
├── 地基 (17种损坏情形)
├── 基础 (6种)
上部承重结构/
├── 混凝土柱 (22种)
├── 砖柱 (20种)
├── 砖墙 (22种)
├── 混凝土梁 (21种)
├── 混凝土板 (20种)
├── 屋架 (19种)
围护结构承重构件/
├── 砌体自承重墙 (19种)
├── 承担水平荷载的填充墙 (19种)
├── 门窗洞口过梁 (15种)
├── 挑梁 (14种)
其他/
├── 非承重墙
├── 内抹灰
├── 外抹灰
├── 楼地面
├── 屋面
├── 顶棚
└── 门窗
```

每条损坏情形记录包含：
- **构件轴线位置**（如 "1-10-A-F轴"、"三层梁 4-C-D轴"）
- **损坏构件查勘情况**（标准化描述 + 可填写数值的空格，如裂缝宽度 __mm）
- **评定结果**（预置）
- **评定标准条款**（预置引用）

### 2.2 输出：《房屋安全鉴定书》

鉴定书包含以下章节：

| 章节 | 内容 | 数据来源 |
|------|------|---------|
| 封面 | 鉴定编号、房屋名称 | 小程序采集 |
| 鉴定机构 | 湖北省建筑工程质量监督检验测试中心有限公司 | 默认值（可改） |
| 查询方式 | URL + 微信二维码 + 查询说明 | 固定内容 |
| **一、房屋概况** | 委托单位/人、房屋坐落、产权信息、建成年份、建筑面积、设计/施工/监理单位、用途、鉴定类别/目的/依据标准、使用历史、周边环境、结构概况 | 大部分来自查勘表 A/B/C/D 模块，小部分 AI 生成 |
| **二、地基及损坏结构构件评定表** | 按构件逐条列出：构件名称、部位、查勘情况、评定结果、标准条款 | 查勘表 E 模块（AI 汇总归类） |
| **三、结构检测成果表** | 检测单位/资质、检测人员、检测内容/依据、主要检测成果（含倾斜观测等） | 小程序采集 + AI 推理总结 |
| **四、鉴定结论及处理建议** | 损坏综述、原因分析、鉴定结论（安全等级）、处理建议、有效期声明 | **核心 AI 推理段落** |
| **五、鉴定房屋照片** | 6 张现场照片 | 小程序采集 |
| 签章栏 | 鉴定人、审核人、签发人、签发时间、单位公章 | 人工/平台填入 |

## 3. 功能需求

### F1. 查勘数据采集（"小程序采集"部分）

> 对应现有 `DataCollection` 页面的 `BuildingProfileForm` + `DamageComponentList` 组件

**F1.1 房屋概况采集**

需完整支持查勘表 Table 0–1 的全部字段，包括：
- 委托方信息（单位、联系人、电话）
- 房屋基本信息（名称、座落、街道/社区、产权人/使用人、产权性质、权证编号）
- 工程参数（建筑面积、总高、层数、层高、檐口高度、鉴定面积）
- 时间信息（建成年份、委托日期、查勘完成日期）
- 参建单位（勘察/设计/施工/监理单位）—— 每项支持"不详"勾选
- 用途信息（原设计用途、现用途）—— 多项 checkbox 勾选
- 结构与构造（结构类型、基础型式、地基处理、墙体类型、楼面型式、屋面构造、楼梯类型、抗震设防烈度）
- 历史变更（用途变更/改造扩建/加固修缮/灾害 —— 每项"有/无/不详"三选一）
- 使用状况（拆改结构/乱搭乱盖/超载使用/其他 —— 每项"有/无/不详"三选一）
- 专项标识（优保建筑/农村危房改造/校外培训机构/自建房专项报告/自建房/经营性自建房/农村危房维修改造/历史遗留办证 —— 是/否）
- 自建房排查编码、普查房屋编号
- 证件图纸资料（多选）
- 周边环境描述
- 现场草图上传

**F1.2 损坏构件查勘数据采集**

对应查勘表 Tables 2–27，核心交互：
- 选择**构件类型**（18 类中的一类或多类）
- 填写**构件轴线位置**（自由文本，如 "三层梁 4-C-D轴"）
- 从该构件类型预置的**损坏情形列表**中勾选匹配项（可多选）
- 对于含数值占位符的项（如裂缝宽度 ___mm），填写实测数值
- 拍摄/上传**现场照片**（关联到具体构件）
- 系统自动带出对应的**评定结果**和**标准条款**

> 现状：数据库中已有 333 条 `evaluation_standard` 记录，`DamageComponent` 已支持 name/category/axisLine/checkedItemIds/photos 字段。需扩展支持自定义查勘描述文本和数值填充。

### F2. AI 智能推理（"AI推理/AI填写"部分）

> 对应现有 `ai-text-generation` 模块 + AI 插件调用

**F2.1 单构件评定推理**

输入：损坏构件勾选的查勘情况列表
输出：综合评定结果 + 适用的评定标准条款

> 对于只勾选一项的简单情况，直接采用该条预置结果。对于勾选多项（含不同严重程度）的情况，AI 综合判断取最不利等级。

**F2.2 房屋损坏情况综述**

输入：全部构件的查勘与评定数据
输出：200–400 字的损坏情况综述文本，按地基基础 → 上部承重结构 → 围护结构 → 其他的逻辑组织。

**F2.3 房屋损坏原因分析**

输入：损坏综述 + 房屋结构类型/建成年份/周边环境/历史变更等上下文
输出：专业的原因分析文本（如材料老化、施工缺陷、环境侵蚀、地基沉降等）。

**F2.4 鉴定结论及处理建议**

输入：全部评定数据 + 损坏综述 + 原因分析
输出：
- **安全等级**：A级（完好）/ B级（基本完好）/ C级（局部危险）/ D级（整体危险）
- **综合结论**：参照鉴定书模板结论段落，包含评定依据、有效期声明
- **处理建议**：正常使用维护 / 维修加固 / 停止使用 / 拆除重建 等，附具体维修建议

**F2.5 鉴定类别说明与鉴定目的**

输入：委托方信息 + 房屋用途 + 专项标识
输出：鉴定类别（整幢鉴定/局部鉴定/专项鉴定）及说明；鉴定目的（如：房屋安全状况评定、危房排查等）。

**F2.6 委托人提供资料整理**

输入：证件图纸资料勾选项
输出：规范化的资料清单文本。

**F2.7 结构检测成果总结**

输入：结构现状检查结果 + 倾斜观测数据
输出：结构检测成果书面总结。

**F2.8 房屋地质勘察及结构补充描述**

输入：结构类型/基础型式/地基处理/层数/檐口高度/楼盖型式/屋盖型式
输出：一段专业的房屋结构概况补充描述文本。

### F3. 鉴定书生成与导出

> 对应现有 `ReportPreview` 页面 + `server/modules/report/`

**F3.1 Word 文档生成**

- 基于鉴定书模板（`.docx`），使用 `docx` 库（项目已集成 v9.7.0）填充字段
- 模板占位符分为三类：
  - `{{key}}` — 直接替换（房屋名称、地址等标量值）
  - `{{#each items}}` — 循环渲染（构件评定表格行）
  - 图片占位符 — 替换为现场照片/立面图/平面示意图

**F3.2 构件评定表生成**

将查勘数据按鉴定书模板 Table 2 格式组织：
- 每类构件生成一组表格行
- 同类型多个构件逐行列出
- 无损构件合并显示（如"二~四层 1-10-A-F轴 除上述损伤构件外其他区域 — 平直牢固，无损伤"）

**F3.3 各层平面及局部损坏示意图**

- 上传的各层平面图 + 标记损坏位置的示意图
- 正立面/背立面照片

**F3.4 导出格式**

- Word（.docx）— 主要导出格式
- PDF — Word 生成后转换

**F3.5 "不详"字段处理**

当采集端勾选"不详"时，鉴定书中对应字段显示"不详"而非留空。适用字段：产权性质、权证编号、建成年份、原勘察/设计/施工/监理单位。

### F4. 原始记录生成

> 对应现有 `OriginalRecordPreview` 页面

基于查勘表的数据，回填生成一份完整的《房屋安全鉴定现场查勘表》Word 文档，作为原始工作底稿存档。要求保持与纸质查勘表一致的表格布局。

**AI 需完成的映射推理：**

1. **去重归类**：同一构件的多条勾选项合并为一条评定记录

## 6. 技术要点

### 6.1 AI 推理 Prompt 设计

需要至少 3 类 Prompt：

1. **房屋概况 AI**（鉴定类别说明、鉴定目的、地质补充描述、资料清单整理）— 输入结构化数据，输出文本段落
2. **构件评定 AI**（单构件综合评定、部位描述规范化）— 输入 checkedItemIds + 数值，输出评定结果 + 标准条款
3. **结论生成 AI**（损坏综述 → 原因分析 → 安全等级 → 处理建议）— 输入全部构件评定 + 房屋概况，输出 JSON（conclusion, riskLevel, suggestion, damageSummary, causeAnalysis）

### 6.2 docx 模板设计

不直接修改原 .docx 文件为模板，而是用 TypeScript + `docx` 库程序化构建文档，优势：
- 动态表格行（构件评定行数不固定）
- 图片动态插入
- 格式一致性好
